"Sharing-Buttons" auf meiner Website

Problem

Ich habe lange Zeit auf meiner Website unter Beiträgen 'Sharing-Buttons' verwendet. Standardmäßig konnte ich dort bei meinem Wordpress Plugin aus einer Liste proprietärer Plattformen auswählen. Lesende meiner Beiträge nkonnten dann z.B. auf das Twitter- oder ein anderes Social Media Symbol klicken - und bekamen einen vorbereiteten Tweet mit Link zu dem Beitrag darin. Blöd ist daran, dass das Fediverse in der Regel nicht mit dabei ist. Eine selbstgebastelte Ergänzung, wie ich das mit den Verlinkungen zu meinen Social Media Profilen gemacht habe, ist hier nicht so schnell möglich, weil es ja nicht nur einen Link zum Teilen gibt, sondern immer erst die genutzte Plattform und Instanz angegeben werden muss.

Meine Recherche und Überlegungen:

Ich habe mir unterschiedliche Lösungen für Sharing-Buttons für Mastodon bzw. das Fediverse angesehen. Am niederschwelligsten scheint mir Toot zu sein. Hier kann ich einfach einen Link generieren, der Nutzende dann dazu auffordert, ihre Instanz einzugeben - danach können sie den Beitrag tooten.

Beispiel

<a href="https://toot.kytta.dev/?text=Blogbeitrag%20https%3A%2F%2ebildungslabor.de">Teile diesen Beitrag auf Mastodon</a>

Ergebnis: Teile diesen Beitrag auf Mastodon

Soweit so gut. Damit wäre es relativ einfach möglich, Mastodon bei den Sharing Buttons zu ergänzen.

Ich habe mir aber überlegt, ob das in solch einer klassischen Form überhaupt sinnvoll ist - oder ich das eher nur deshalb suche, weil 'man' das eben so macht. Ich kam zum Ergebnis, dass ich gerne möchte, dass Inhalte von mir für andere hilfreich sind. Das gelingt aber sicherlich nicht, indem ich alle dazu auffordere sie mit vorformulierten Texten ins Internet zu posaunen. Außerdem deckt eine begrenzte Auswahl an Sharing-Buttons niemals die Vielfalt im Internet ab. Wenn ich doch eine bestimmte Auswahl anbiete, formuliere ich damit indirekt immer auch eine Norm, was 'im Internet' der vermeintliche Standard ist.

Gefundene Lösung

Die für mich bessere Lösung statt klassischer Sharing-Buttons ist deshalb ein Button, mit dem sich der Beitragslink kopieren lässt. Das ist dann keine Aufforderung im Sinne von 'Bitte verbreite meinen Content auf einer der x angebotenen Plattformen', sondern eine Einladung bzw. Ermöglichung: 'Mache mit diesem Link, was Du möchtest'. Menschen können den Link dann also für sich abspeichern, mit ihren eigenen Anmerkungen in sozialen Netzwerken teilen oder auch direkt an Kolleg*innen weitergeben. Nebenbei löst sich das Problem mit der potentiellen Kompliziertheit eines Fediverse-Sharing-Buttons, denn den brauche ich jetzt ja gar nicht mehr :-)

Mein Vorgehen

Um einen 'Kopiere den Beitragslink'-Button unter jeden Beitrag einzufügen, bin ich folgendermaßen vorgegangen:

  • Ich habe mir diesen Code geklaut und leicht angepasst - und zwar so:

    • 
<html>
<body>
<p>
<hr>
</p>
<h5>
Beitrag merken & teilen
</h5>
<p>
Hier kannst Du dir den Link zum Beitrag kopieren - beispielsweise um ihn für Dich zu speichern oder mit anderen zu teilen.
</p>
<button onclick="copyToClipboard()">Link kopieren

<script>
function copyToClipboard(text) {
var inputc = document.body.appendChild(document.createElement("input"));
inputc.value = window.location.href;
inputc.focus();
inputc.select();
document.execCommand('copy');
inputc.parentNode.removeChild(inputc);
alert("Du hast den Beitragslink kopiert und kannst ihn nun dort nutzen bzw. teilen, wo Du möchtest.");
}
</script>

</html>

  • Ich habe das WPCode Plugin auf meiner Website - und kann den Code dort in ein Custom-Code HTML-Feld einfügen und dieses aktivieren
  • Für die Sichtbarkeit des Code stelle ich 'Unter jedem Beitragsinhalt' ein.

Fertig! Das Ergebnis kannst Du Dir auf meinem Blog am Ende jedes Beitrags ansehen.
